A_few_bugs_and_solutions_of_Glassfish

2018-06-23

  1. When using Java 8 with Glassfish, you need to make sure glassfish version is new enough (like some nightly version of 5.0.1) to avoid a bug that disables SSL related modules, for example Https and secure admin console.

  2. There could be some errors saying “exec: 58: java not found”. An easy solution is to edit asenv.conf(asenv.bat in windows) in server config folder(glassfish5/glassfish/config/asenv.conf). Simply add the following:
    AS_JAVA="/mnt/efs/jdk1.8.0_172"

  3. Glassfish does not support JDK9 or higher.(till 5.0.1)

  4. some useful command
    asadmin create-domain gsv asadmin delete-domain domain1 asadmin change-admin-password asadmin start-domain asadmin enable-secure-admin asadmin create-service

  5. When using Spring websocket module, there was a bug that Spring cannot get javax.websocket.server.ServerContainer from ServletContext. I cannot tell the exact reason but evidence shows that “asadmin enable-secure-admin” command messed up the domain, caused the issue. Calling “disable-secure-admin” will NOT solve the problem. You need to stop the domain, disable-secure-admin, delete-domain, and then recreate the domain. The newly created domain should have secure-admin disabled by default(not 100% sure so call disable-secure-admin first). Use the new domain and it should work fine.
    See my post reporting this issue to GF for details:
    https://github.com/javaee/glassfish/issues/22533
